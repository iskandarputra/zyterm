# CLI reference

The authoritative catalogue of every zyterm command-line flag, its argument, default, and
behaviour. Generated by reading `src/main.c` directly — the `lo[]` `getopt_long` table
(`src/main.c:406`), every `case` in the parse loop (`src/main.c:463`), and `usage()`
(`src/main.c:83`). When the binary and this page disagree, the binary wins; please file a fix.

zyterm is version **1.2.0** (`ZT_VERSION`, `src/zt_ctx.h:70`).

```
zyterm [OPTIONS] <DEVICE>
zyterm --replay <LOGFILE>
```

`<DEVICE>` is a serial node (`/dev/ttyUSB0`), a `tcp://host:port` raw socket, or a
`telnet://host:port` socket with Telnet IAC escaping. `rfc2217://` is parsed but **not
implemented** — see [Transports](#transports--device-argument) and
[KNOWN_ISSUES](../tracking/KNOWN_ISSUES.md). `<DEVICE>` becomes optional when a discovery hint
(`--port-glob` / `--match-vid-pid`) is supplied, and is unused entirely in `--replay`, `--diff`,
and `--profile-save` modes.

## Connection

| Flag | Argument | Default | Behaviour |
| --- | --- | --- | --- |
| `-b`, `--baud` | `<rate>` | `115200` | Arbitrary baud via termios2. Validated by `parse_baud`: `strtoul` base 10, must be non-zero and ≤ 20000000, else `zt_die` (`src/main.c:280`). |
| `--data` | `<5\|6\|7\|8>` | `8` | Data bits. `atoi`, rejected unless 5–8 (`src/main.c:475`). |
| `--parity` | `<n\|e\|o>` | `n` | none / even / odd; first char, case-insensitive (`src/main.c:481`). |
| `--stop` | `<1\|2>` | `1` | Stop bits; must be exactly 1 or 2 (`src/main.c:487`). |
| `--flow` | `<n\|r\|x>` | `n` | none / RTS-CTS / XON-XOFF. Also accepts `c` (=`r`) and `s` (=`x`) (`src/main.c:493`). |
| `--reconnect` | — | **ON** | Auto-reopen on hang-up. This is the default; the flag is for overriding a profile (`src/main.c:505`). |
| `--no-reconnect` | — | — | Exit on serial hang-up instead of reopening (`src/main.c:506`). |
| `--port-glob` | `<pat>` | — | Re-resolve the device on every reconnect, e.g. `"/dev/ttyUSB*"`. Makes `<DEVICE>` optional (`src/main.c:635`, `:713`). |
| `--match-vid-pid` | `<V:P>` | — | Filter discovered ports by USB vendor:product in **hex**, e.g. `0403:6001` (FT232R), `1a86:7523` (CH340). Each half must fit in 16 bits (`src/main.c:636`). |

Reconnect is on by default; see [ADR-0004](../decisions/0004-reconnect-on-by-default.md) for why.

## Logging & capture

| Flag | Argument | Default | Behaviour |
| --- | --- | --- | --- |
| `-l`, `--log` | `<file>` | — | Append RX log with ms timestamps; opened `O_WRONLY\|O_CREAT\|O_APPEND` (`src/main.c:720`). `Ctrl+A l` toggles an auto-named `zyterm-YYYYMMDD-NNN.txt` at runtime. |
| `--log-max-kb` | `<N>` | — | Rotate to `<file>.1` once the log exceeds N KB. `atol`, must be > 0 (`src/main.c:514`). |
| `--log-format` | `<text\|json\|raw>` | `text` | Log encoding. `json` is NDJSON; `raw` is unframed bytes (`src/main.c:567`). |
| `--tx-ts` | — | off | Also write TX to the log with a `-> ` prefix (`src/main.c:520`). |
| `--mute-dbg` | — | off | Drop `<dbg>`-level lines from both log and scrollback (`src/main.c:613`). |
| `--mute-inf` | — | off | Drop `<inf>`-level lines from both log and scrollback (`src/main.c:614`). |
| `--dump` | `<sec>` | — | Headless capture for N seconds (`0` = forever); no interactive UI. Negative is clamped to 0 (`src/main.c:471`). See [modes](#exit--non-interactive-modes). |
| `--replay` | `<file>` | — | Replay a capture through the UI; no device opened (`src/main.c:521`, `:686`). |
| `--replay-speed` | `<x>` | `1.0` | Replay speed multiplier; `atof`, `0` means as-fast-as-possible, negatives clamped to 0 (`src/main.c:522`). |
| `--rec` | `<file>` | — | Record the session as an asciinema cast v2 file; play back with `asciinema play <file>` (`src/main.c:649`). |

See the [logging-and-capture guide](../guide/logging-and-capture.md) for workflows.

| --- | --- | --- | --- |

## Advanced I/O

| Flag | Argument | Default | Behaviour |
| --- | --- | --- | --- |
| `--threaded` | — | off | Drain the serial fd on a worker thread into a lock-free SPSC ring; reduces UART-latency jitter at ≥ 1 Mbaud (`src/main.c:615`, `src/loop/rx_thread.c`). |
| `--autobaud` | — | off | Probe common rates after open and pick the one with the highest printable-ASCII ratio. Ignores `-b`; sets `baud = 0` as a sentinel, then opens at 115200 before probing (`src/main.c:616`, `:736`). Re-runnable at runtime via `Ctrl+A A`. |

> The `--epoll` "fast path" flag was **removed in 1.2.0**; the epoll/splice code in
> `src/serial/fastio.c` is unwired and the runtime uses `poll(2)`. See
> [ADR-0003](../decisions/0003-epoll-splice-fastpath-deferred.md).

## HTTP & streaming

| Flag | Argument | Default | Behaviour |
| --- | --- | --- | --- |
| `--http` | `<port>` | — | Start the built-in HTTP server (HTML UI at `/`, SSE at `/stream`, WebSocket at `/ws`, `GET /api/state`, `POST /api/send`, `GET /metrics`). **⚠ Parsed with `atoi`, truncated to a `uint16`, no range validation** (`src/main.c:586`) — see [ZT-020](../tracking/KNOWN_ISSUES.md). |
| `--webroot` | `<dir>` | — | Serve static files from `<dir>`; `..` traversal is blocked (`src/main.c:587`). |
| `--http-cors` | — | off | Add `Access-Control-Allow-Origin: *` headers (`src/main.c:591`). |
| `--metrics` | `<path>` | — | Open an `AF_UNIX` socket at `<path>` emitting Prometheus text (`src/main.c:582`). |

> The HTTP bridge is an **unauthenticated** local trust boundary. `POST /api/send` writes to the
> serial line with no auth and the WS upgrade does no Origin check — see
> [ZT-004 / ZT-013](../tracking/KNOWN_ISSUES.md) before exposing it beyond loopback. Background:
> [design/HTTP_BRIDGE](../design/HTTP_BRIDGE.md).

## Sessions & integration

| Flag | Argument | Default | Behaviour |
| --- | --- | --- | --- |
| `--detach` | `<name>` | — | Expose the live session on an `AF_UNIX` socket so another zyterm can attach (`src/main.c:592`, `:750`). |
| `--attach` | `<name>` | — | Attach to a detached session as a read-write tty; **exits when the session ends** without opening a device (`src/main.c:596`). |
| `--filter` | `<cmd>` | — | Pipe RX through `sh -c <cmd>` before display, e.g. `--filter 'grep -v noise'` (`src/main.c:578`, `:749`). |
| `--diff` | `<a> <b>` | — | Diff two capture files and exit; consumes the next positional arg as `<b>` (`src/main.c:619`). See [modes](#exit--non-interactive-modes). |

> The detach socket is `/tmp/zyterm.<name>.sock` with default umask and no peer-credential check —
> a local trust boundary; see [ZT-012](../tracking/KNOWN_ISSUES.md).

## Display & input

| Flag | Argument | Default | Behaviour |
| --- | --- | --- | --- |
| `-x`, `--hex` | — | off | Render RX as a hex dump (`src/main.c:467`). Toggle at runtime with `Ctrl+A h`. |
| `-e`, `--echo` | — | off | Start with local echo on (`src/main.c:468`). Toggle with `Ctrl+A e`. |
| `--no-color` | — | colour on | Disable RX log-level colouring (`src/main.c:469`). |
| `--ts` | — | off | Start with timestamp display on (`src/main.c:470`). Toggle with `Ctrl+A t`. |
| `--watch` | `<pattern>` | — | Highlight matching lines. **Repeatable, up to 8** (`ZT_WATCH_MAX`, `src/zt_ctx.h:62`); a 9th triggers `zt_die` (`src/main.c:507`). |
| `--watch-beep` | — | off | Emit BEL on a watch match (`src/main.c:513`). |
| `--macro` | `F<n>=<str>` | — | Bind a macro to F1..F12 (`n` 1–12, `ZT_MACRO_COUNT`). Supports `\r \n \t \xNN` escapes. **Repeatable**; an out-of-range `n` or missing `=` is fatal (`src/main.c:527`). |

Line-ending `<mode>` for `--map-out` / `--map-in`: `none \| cr \| lf \| crlf \| cr-crlf \| lf-crlf`
(parsed by `eol_parse`). For the runtime equivalents, see
[reference/KEYBINDINGS](./KEYBINDINGS.md).

## Profiles

| Flag | Argument | Default | Behaviour |
| --- | --- | --- | --- |
| `--profile` | `<name>` | — | Load `~/.config/zyterm/<name>.conf` at startup, then watch it for edits (Linux inotify) and hot-reload (`src/main.c:601`, `:753`). |
| `--profile-save` | `<name>` | — | Snapshot the current CLI-supplied settings to that profile and **exit** before opening the device (`src/main.c:605`, `:673`). See [modes](#exit--non-interactive-modes). |

Profiles are persisted at `~/.config/zyterm/<name>.conf` — not `~/.zyterm/profiles`. Background:
[ADR-0006](../decisions/0006-in-memory-history-and-bookmarks.md).

## Event hooks

| Flag | Argument | Default | Behaviour |
| --- | --- | --- | --- |
| `--on-match` | `/RE/=CMD` | — | Run shell `CMD` when an RX line matches the POSIX ERE. Use a `send:BYTES` action (with `\r \n \xNN` escapes) to inject TX instead of spawning a shell (`src/main.c:650`). |
| `--on-connect` | `<cmd>` | — | Fire after the device opens, including on every reconnect (`src/main.c:651`). |
| `--on-disconnect` | `<cmd>` | — | Fire when the device closes or hangs up (`src/main.c:652`). |

Hooks receive these environment variables: `ZYTERM_PORT`, `ZYTERM_BAUD`, `ZYTERM_LINE`,
`ZYTERM_PATTERN`. See the [automation guide](../guide/automation.md) for recipes.

## Meta

| Flag | Argument | Behaviour |
| --- | --- | --- |
| `-h`, `--help` | — | Print the help/usage screen to **stderr** (colour-aware; auto-disabled when stderr is not a TTY, or `NO_COLOR`/`TERM=dumb` is set) and exit 0 (`src/main.c:659`). |

## Transports / device argument

The positional `<DEVICE>` (`src/main.c:717`) accepts:

| Form | Meaning |
| --- | --- |
| `/dev/ttyUSB0` | A serial node or any character device. |
| `tcp://host:port` | Raw TCP, e.g. ser2net in raw mode. |
| `telnet://host:port` | TCP with Telnet IAC escaping. |
| `rfc2217://host:port` | **Not implemented** — parsed, then `zt_die` with "use ser2net raw + `tcp://`" (`src/serial/transport.c:94`). |

The `rfc2217://` stub is an intentional deferral; see
[ADR-0005](../decisions/0005-rfc2217-deferred.md) and
[KNOWN_ISSUES](../tracking/KNOWN_ISSUES.md). For first-run walkthroughs see
[getting-started](../guide/getting-started.md).

## Exit / non-interactive modes

Several flags bypass the interactive TUI entirely and return without ever entering
`run_interactive`:

| Flag | What happens | Device opened? |
| --- | --- | --- |
| `--dump <sec>` | Headless capture for N seconds (`run_dump`), then exit (`src/main.c:765`). | Yes |
| `--replay <file>` | Replay the capture through the UI, then exit (`run_replay`, `src/main.c:694`). | No |
| `--diff <a> <b>` | Diff two captures and return `diff_run`'s code (`src/main.c:619`). | No |
| `--profile-save <name>` | Snapshot settings to the profile and exit (`src/main.c:673`). | No |
| `--attach <name>` | Attach to a detached session, return when it ends (`src/main.c:596`). | No |

`--dump` is the only non-interactive mode that opens the serial device; the rest exit before
`setup_serial`. All five run `scrollback_free` before returning so they are safe in embedded
builds (`zyterm_main` re-entrancy resets getopt at `src/main.c:308`).
